How this build works

The API stores metadata, policies, users, devices, packages, and jobs. The actual Windows paths remain where they already are, such as H:\ or any exact folder you choose.

Direct local path work and SMB publishing are handled by the Windows Host Bridge layer, not by moving your files into Docker.

Role

Owns all local path work on Windows so Docker stays isolated from your real disks.

SMB Path Publishing

Maps approved shares to Windows SMB only when policy allows it, instead of exposing entire drives blindly.

Install-Only Gate

Future agent jobs execute installers from approved entry paths without turning every package into a normal downloadable share.

Telemetry

Feeds the API with device heartbeats, scan results, publish state, and future copy/install timings.
